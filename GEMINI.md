# Project Context: Trading Analysis

## Infrastructure & Deployment

- **Region**: Consolidated in **Tokyo (`ap-northeast-1`)**.
- **CI/CD**: Native **AWS CodePipeline** + **CodeBuild**, triggered by GitHub `main` branch.
- **Access**: Stable **Application Load Balancer (ALB)** URL.
- **Outbound**: Single **NAT Gateway** for a stable outbound Static IP.
- **Dagster Version**: **1.11** (Modern UI).

## Architecture

| Component | AWS Resource | Details |
| :--- | :--- | :--- |
| **Access** | **ALB** | `http://trading-analysis-v2-1794112824.ap-northeast-1.elb.amazonaws.com` |
| **Orchestrator** | **ECS Fargate** | Dagster Webserver & Daemon (Runs `trading_dagster` module) |
| **Metadata DB** | **RDS Postgres** | `db.t4g.micro` - Stores run history & event logs |
| **Analytics DB** | **ClickHouse Cloud** | External managed service (Direct ingestion from Dagster) |
| **Networking** | **NAT Gateway** | Provides a single **Static Public IP** for ClickHouse allow-listing |
| **Data Lake** | **S3** | Raw data storage (`trading-analysis-data-v2`) |
| **Visualization**| **ECS Fargate** | Grafana (Anonymous Admin access, pre-loaded dashboards) |

## Key Configurations & Automation

- **Direct Ingestion**: Data is polled from Binance every 5-10 minutes and inserted directly into ClickHouse Cloud.
- **Auto-generated Secrets**: RDS passwords are generated by Terraform and automatically injected into Secrets Manager.
- **ECR Public**: Docker builds use `public.ecr.aws` base images.
- **VPC Design**: Services run in private subnets; access is strictly via ALB or NAT Gateway.
