# Project Context: Trading Analysis

## Infrastructure & Deployment

- **Region**: Consolidated in **Tokyo (`ap-northeast-1`)**.
- **CI/CD**: Native **AWS CodePipeline** + **CodeBuild**, triggered by GitHub `main` branch.
- **Access**: Stable **Application Load Balancer (ALB)** URL.
- **Outbound**: Single **NAT Gateway** for a stable outbound Static IP.

## Architecture

| Component | AWS Resource | Details |
| :--- | :--- | :--- |
| **Streaming Producer** | **ECS Fargate** | `ws-consumer` — Binance WebSocket → Kafka/Redpanda topic `binance.price.ticks` |
| **Message Bus** | **ECS Fargate** | Redpanda (Kafka-compatible) |
| **PnL Consumers** | **ECS Fargate** | `pnl-consumer-{prod,bt,real-trade,price}` — Kafka → real-time PnL → ClickHouse |
| **Batch Recompute** | **`scripts/audit_pnl.py`** | Standalone window delete+recompute / audit (prod, bt, real_trade) |
| **Market-Data Backfill** | **AWS Lambda** | `backfill_prices` — daily ccxt gap-fill of `futures_price_1min` |
| **Analytics DB** | **ClickHouse Cloud** | External managed service; 1hour/1day rollups via Materialized Views |
| **Networking** | **NAT Gateway** | Provides a single **Static Public IP** for ClickHouse allow-listing |
| **Data Lake** | **S3** | Raw data storage (`trading-analysis-data-v2`) |
| **Visualization**| **Grafana Cloud** | L1-L5 PnL Dashboards (deployed via CI) |

## Key Configurations & Automation

- **Real-time Ingestion**: Binance WebSocket candles flow through Redpanda; the `price` pnl-consumer writes `futures_price_1min` from `candle.open`.
- **Window Repair**: `scripts/audit_pnl.py --fix-window` does idempotent delete+recompute of a PnL window and rebuilds the rollups.
- **Auto-generated Secrets**: Secrets are generated by Terraform and automatically injected into Secrets Manager.
- **ECR Public**: Docker builds use `public.ecr.aws` base images.
- **VPC Design**: Services run in private subnets; access is strictly via ALB or NAT Gateway.
