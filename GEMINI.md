## Architecture

| Component | AWS Resource | Role |
| --- | --- | --- |
| **Orchestrator** | ECS Fargate | Runs Dagster Webserver & Daemon |
| **Metadata DB** | RDS Postgres | Stores Dagster run history and logs |
| **Analytics DB** | ClickHouse Cloud | External managed service for trade data |
| **Stream Buffer** | MSK Serverless | Kafka interface for real-time ingestion |
| **Data Lake** | S3 | Raw data storage (`trading-analysis-data-v2`) |
| **Visualization**| ECS Fargate | Runs Grafana for PnL dashboards |

## Deployment Workflow

1.  **CodePush**: Push to GitHub `main` branch.
2.  **CodePipeline**: Automatically triggers in Tokyo.
3.  **Test Stage**: Runs `pytest`.
4.  **Build Stage**: Docker build (via ECR Public) and push to local ECR.
5.  **Deploy Stage**: 
    - Terraform apply (Infrastructure updates).
    - ECS Service update (Force new deployment).
    - Database schema sync (ClickHouse).

## Security Note

- **CodeBuild IAM**: The `trading-analysis-codebuild` role is granted `AdministratorAccess` in the Terraform code to ensure it can manage MSK, RDS, and ECS without permission deadlocks.
- **RDS Password**: Automatically generated by Terraform and stored in Secrets Manager (`trading-analysis/dagster-pg`).
