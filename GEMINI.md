# Project Context: Trading Analysis

## Infrastructure & Deployment

- **Region**: Consolidated in **Tokyo (`ap-northeast-1`)**.
- **CI/CD**: Native **AWS CodePipeline** + **CodeBuild**, triggered by GitHub `main` branch.
- **Access**: Stable **Application Load Balancer (ALB)** URL.
- **Dagster Version**: **1.11** (Modern UI).

## Architecture

| Component | AWS Resource | Details |
| :--- | :--- | :--- |
| **Access** | **ALB** | `http://trading-analysis-165456032.ap-northeast-1.elb.amazonaws.com` |
| **Orchestrator** | **ECS Fargate** | Dagster Webserver & Daemon (Runs `trading_dagster` module) |
| **Metadata DB** | **RDS Postgres** | `db.t4g.micro` - Stores run history & event logs |
| **Analytics DB** | **ClickHouse Cloud** | External managed service (Trade data storage) |
| **Stream Buffer** | **Amazon Kinesis** | Cost-effective alternative to MSK for real-time ingestion |
| **Data Lake** | **S3** | Raw data storage (`trading-analysis-data-v2`) |
| **Visualization**| **ECS Fargate** | Grafana (Provisioned with ClickHouse datasource) |

## Key Configurations & Automation

- **Auto-generated Secrets**: RDS passwords are generated by Terraform and automatically injected into Secrets Manager (`trading-analysis/dagster-pg`).
- **ECR Public**: Docker builds use `public.ecr.aws` base images to avoid Docker Hub rate limits.
- **Subpath Routing**: ALB routes `/grafana*` to Grafana and everything else to Dagster. Grafana is configured with `GF_SERVER_SERVE_FROM_SUB_PATH=true`.
- **IAM Security**: The CodeBuild service role has specific service-level permissions to ensure Terraform can manage the entire stack without manual intervention.
